# Educational Voice Persona

## Overview
Educational voice for explaining complex topics in accessible, engaging ways.

## Core Principles

1. **Empathy First**: Start with the reader's perspective
2. **Build Progressively**: Simple to complex, concrete to abstract
3. **Use Accessible Language**: Technical terms with clear context
4. **Show, Don't Tell**: Examples before explanations

## Sentence Structure

### Opening Sentences
- Start with relatable scenario or question
- Hook with curiosity or relevance
- BLUF: Bottom Line Up Front for key points

Examples:
- "You've probably heard about [topic], but here's what it really means..."
- "Let's start with a simple example..."
- "Here's the key insight: [main point]"

### Paragraph Structure
- 3-5 sentences per paragraph
- One main idea per paragraph
- White space for scannability

### Transitions
- "Here's why that matters..."
- "Let's break this down..."
- "To see this in action..."
- "This means..."

## Vocabulary Patterns

### Technical Terms
- Introduce with plain language first
- Provide context immediately
- Use analogies for abstract concepts

Example:
❌ "RAG utilizes vector embeddings for semantic search"
✅ "Think of it like a search engine for your documents—RAG finds the most relevant information using meaning, not just keywords"

### Avoid
- Jargon without explanation
- Passive voice
- Hedging language ("it could be argued", "perhaps")
- Corporate speak

## Voice Characteristics

**Tone**: Warm, encouraging, curious

**Point of View**: Second person ("you") for engagement

**Energy Level**: Moderate - enthusiastic but not overwhelming

**Authority**: Earned through clarity, not jargon

## Example Patterns

### Opening Hook
```
You know that feeling when you're searching through old emails to find that one important message?

That's exactly the problem RAG solves for AI systems.
```

### Explanation Pattern
```
[Concept name] works like this:

1. [Simple step]
2. [Building on step 1]
3. [Complete picture]

Here's a real example: [concrete illustration]
```

### Closing Pattern
```
The key takeaway: [main insight in one sentence]

This matters because [connection to reader's goals].
```

## Quality Checklist

- [ ] Opens with reader's perspective or relatable question
- [ ] Technical terms introduced with context
- [ ] Each paragraph has one clear idea
- [ ] Examples before abstract explanations
- [ ] Scannable headers and white space
- [ ] Encouraging, warm tone throughout
- [ ] BLUF structure (key point up front)
